CRITICAL REQUIREMENTS:

1. SAT AUTHENTICITY (HIGHEST PRIORITY):
   - SAT probability questions are NEVER straightforward textbook exercises
   - Every question must include at least ONE of these SAT-style elements:
     * An unexpected twist or counterintuitive setup
     * Multiple steps requiring careful logical sequencing
     * A scenario requiring interpretation before calculation
     * Integration of probability with another math concept
   - AVOID questions that can be solved by direct formula application with no interpretation
   - AVOID overly simple scenarios like "What is the probability of drawing one red marble from a bag?"
   - Use sophisticated contexts: medical testing accuracy, survey sampling bias, quality control with multiple stages, experimental design, risk assessment, game theory scenarios, genetic inheritance, data reliability

2. DIFFICULTY CALIBRATION - BE HONEST:
   - Difficulty 1-2: Single concept, 1-2 steps, minimal interpretation (rarely appropriate for SAT)
   - Difficulty 3: Two concepts or 2-3 reasoning steps with some interpretation required
   - Difficulty 4: Multiple concepts, conditional thinking, or non-obvious setup requiring 3-4 steps
   - Difficulty 5: Complex conditional probability, multiple dependencies, or requires recognizing non-obvious structure
   - Count actual reasoning steps honestly—don't inflate difficulty for basic multiplication
   - Simple "without replacement" scenarios are typically difficulty 2-3, not 4-5

3. DISTRACTOR DESIGN (CRITICAL - MOST COMMON FAILURE POINT):
   - NEVER use generic nearby values or arbitrary numbers
   - Each distractor must show the complete wrong calculation in your rationale
   - Target these specific error patterns (vary across questions):
     * Treating dependent events as independent: using P(A) × P(B) when replacement matters
     * Using original probabilities after an event changes the sample space
     * Confusing P(A|B) with P(B|A) or P(A and B)
     * Calculating 1 - P(event) incorrectly for complements
     * Adding probabilities instead of using P(A or B) = P(A) + P(B) - P(A and B)
     * Forgetting to update denominators in "without replacement" scenarios
     * Using permutations when order doesn't matter, or vice versa
     * Confusing "at least one" with "exactly one"
   - In your distractor rationales, write: "A student who [specific error] would calculate: [step 1], [step 2], arriving at [this choice]"
   - DO NOT write vague rationales like "confuses independent and dependent events" without showing the calculation

4. ANSWER CHOICES:
   - All four choices must be numerically distinct after simplification
   - Verify no two fractions reduce to the same value
   - Choices should emerge naturally from the error patterns you've identified
   - Avoid patterns like {correct answer, 2×correct, ½×correct, complement} unless each represents a genuine misconception

5. SCENARIO VARIETY:
   - Avoid repetitive setups across questions you generate
   - Do not default to marble/ball selection from containers
   - Use diverse, realistic contexts that feel like actual SAT questions

6. SAT FORMATTING:
   - Use College Board's concise, precise wording
   - Clear stem, four labeled choices (A-D)
   - Avoid unnecessary wordiness or overly casual language

7. RATIONALES:
